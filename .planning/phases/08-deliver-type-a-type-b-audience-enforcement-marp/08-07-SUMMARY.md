## Decisions Made

1. **Single-quoted bash arg + F_PATH env var** for the inline node-eval (instead of double-quoted with shell interpolation). Rationale: bash double-quote consumes `\s` to literal `s`, breaking regex matches. Discovered when Tests 5a/5b failed with `MISSING: <all 5 fields>`. Single-quote preserves backslash escapes; F_PATH env var is also shell-injection-safe.
2. **Validator handles BOTH nested + flat-dotted YAML forms in a single pass** — two regex matchers per field, OR'd together. No need for a YAML parser dependency or preprocessor pass.
3. **marp-environment.md includes Firefox** alongside Chrome/Edge — RESEARCH 1527-1543 mentions Chrome/Edge only; Firefox is a Marp-supported browser per upstream docs (puppeteer-core supports both Chromium and Firefox channels) and adds installed-base coverage on Linux desktops.
4. **Pandoc fallback documented but explicitly NOT auto-invoked** — preserves Marp-quality bar; Pandoc PPTX is significantly less polished. User can run pandoc manually as escape hatch.
5. **hooks/dist/brief-validate-frontmatter.sh regenerated but NOT committed** — `hooks/dist/` is gitignored; regenerated by `npm run build:hooks` at install time. Source-of-truth lives at `hooks/brief-validate-frontmatter.sh`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Bash double-quote consumed regex backslash escapes in inline node-eval**

- **Found during:** Task 2 sub-task A (initial GREEN test run)
- **Issue:** The inline node-eval validator was written with bash double-quoted arg (`node -e "..."`). Bash double-quote interprets `\s` as `s` (since backslash followed by an unrecognized character drops the backslash inside double quotes). Result: `new RegExp('^audience:\s*$')` arrived at node as `new RegExp('^audience:s*$')` — matched literal `s` instead of whitespace, so all 5 mandatory fields appeared "missing" even when present. Tests 5a and 5b caught this (2 of 14 tests failed; the other 12 passed because they don't depend on the regex actually matching valid frontmatter).
- **Fix:** Switched the node -e arg from double-quoted to single-quoted, replaced the embedded JSON-style double-quotes with the shell-safe pattern (single quotes around the script, double quotes inside the script for JS string literals). File path was already passed via `F_PATH` env var (no shell interpolation needed in the script body).
- **Files modified:** `hooks/brief-validate-frontmatter.sh` (validator block lines 52-89)
- **Verification:** All 14 tests pass after the fix; manual re-run with `bash hooks/brief-validate-frontmatter.sh` against handcrafted fixtures confirmed expected OK / MISSING / NO_FRONTMATTER outputs.
- **Committed in:** `ebec28c` (Task 2 GREEN commit — fix is part of the same atomic implementation, not a separate commit, since the bug was caught and fixed before the GREEN commit was created)
